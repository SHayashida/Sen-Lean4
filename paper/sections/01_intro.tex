\section{Introduction}

Classical impossibility theorems explain why certain axiom sets cannot hold simultaneously, but engineering workflows also need the complementary question: \emph{which minimal lever changes restore feasibility}.
This paper frames that question as an auditable systems problem.

Our central thesis is practical: a small, proof-carrying atlas can already provide publishable insight if it fixes one clear chain of trust.
In this paper, the golden thread is \emph{auditable verification}: external SAT search is allowed to be heuristic and untrusted, but the final evidence must reduce either to Lean proofs or to replayable, independently audited artifacts.
We operationalize this thesis in sen24 with claim blocks C1--C6 mapped to concrete commands and artifacts in \texttt{docs/paper\_claims\_map.md}.

\paragraph{Big Three Contributions.}
\begin{itemize}[leftmargin=*]
  \item \textbf{Hybrid verification architecture.} We separate Lean semantics and theorem statements from SAT search, then attach LRAT certificates so that final UNSAT claims are checked by the Lean kernel rather than trusted to the solver.
  \item \textbf{Auditable CNF pipeline.} We pair DIMACS generation with manifests, independent auditing, clause-family invariants, and replay commands, so encoding drift is detectable instead of implicit.
  \item \textbf{Sen24 as dual evidence.} The fixed base case provides both a constructive Lean-side impossibility witness and a committed proof-carrying UNSAT instance, giving one semantic proof path and one exhaustive certificate path for the same small scope.
\end{itemize}

\paragraph{Official Claim Ladder.}
The paper-level claims follow the current official map exactly:
\begin{itemize}[leftmargin=*]
  \item \textbf{C1}: sen24 axiom-lever enumeration yields a reproducible SAT/UNSAT frontier.
  \item \textbf{C2}: UNSAT boundary cases are explainable via one MUS and one small MCS candidate.
  \item \textbf{C3}: UNSAT evidence can be attached as LRAT and checked in Lean.
  \item \textbf{C4}: symmetry reduction and monotone pruning are used only under explicit assumptions and runtime validators.
  \item \textbf{C5}: relaxed SAT cases yield a deterministic, auditable gallery with explicit SAT witness validation.
  \item \textbf{C6}: repair enumeration is checked against an independent optimum baseline by triangulation.
\end{itemize}

This ladder is intentionally conservative. It distinguishes what the repository can produce from what each revision re-verifies, and it does not broaden sen24 evidence into a claim about arbitrary $n,m$.
