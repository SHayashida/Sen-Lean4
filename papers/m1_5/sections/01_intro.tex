\section{Introduction}

The repository now supports two manuscript tracks inside one shared research codebase.
The first paper (M1) remains under \code{paper/} and continues to use the established asset and reproducibility workflow.
The second paper (M1.5) uses the dedicated workspace \code{papers/m1_5/} so that writing, figures, tables, and claim notes can evolve independently while still drawing on the same underlying Lean, SAT, and atlas infrastructure.

This separation is structural rather than branch-based.
Shared code and reusable data continue to evolve on the common repository trunk, and paper submissions are expected to be frozen by commit hash and tag rather than by long-lived manuscript branches.
