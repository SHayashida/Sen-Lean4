\section{Reproducibility Note}
\label{sec:repro}

This manuscript is grounded in a kernel-checked Lean module and repository-local reproducibility artifacts rather than new solver experiments.
The main evidence sources used in the text are:

\begin{itemize}[leftmargin=1.5em]
\item \code{SocialChoiceAtlas/Sen/Lifting/ImpossibilityLift.lean} --- the positive flagship theorem \(\mathtt{sen\_impossibility\_lifts}\) and its supporting helpers.
\item \code{SocialChoiceAtlas.lean} --- the root import line that wires the lifting module into the default \(\mathtt{lake\ build}\).
\item \code{SocialChoiceAtlas/Sen/BaseCase24/Theorem.lean} --- the read-only source of the ported \(\mathtt{sen\_not\_acyclic\_01}\) case-split that the lift re-states polymorphically.
\item \code{docs/m2\_scope\_wall.md} --- the scope wall.
\item \code{docs/no\_cycle\_interpretation\_note.md} and \code{docs/axiom\_semantics\_scaling.md} --- the prior repository notes that establish why the short-cycle encoding does not transfer past \(m = 4\).
\item \code{scripts/check\_acyclicity\_short\_cycles.py} --- the finite check that audits the short-cycle encoding at the base case.
\end{itemize}

\subsection{Build commands}
\label{sec:build-commands}

The narrow build command that exercises the positive flagship is:
\begin{quote}
\code{lake build SocialChoiceAtlas.Sen.Lifting.ImpossibilityLift}
\end{quote}

The full default kernel build is:
\begin{quote}
\code{lake build}
\end{quote}

The manuscript itself builds with:
\begin{quote}
\code{make -C papers/m2 pdf}
\end{quote}
